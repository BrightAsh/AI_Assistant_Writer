# AI_Assistant_Writer
> 메인 작가의 업무를 보조하는 AI 기반 보조 작가 시스템

##  Role
- Character Style Consistency Checker
  - 캐릭터 대사에서 말투와 개성의 일관성을 분석하는 AI 기반 보조 작가 시스템  
  > **Prompt Engineering 기반 설계 및 시스템 구현**<br>
  > **GPT 기반 캐릭터 개성 일관성 평가 시스템 구현**

##  전체 분석 흐름

1. **캐릭터 이름 추출**
   - 대사 속 화자 이름을 정규표현식 기반으로 탐색
   - GPT를 통해 '화자'만을 필터링하며, 불필요하거나 모호한 이름 제거

2. **시점별 장면 분류**
   - 각 Scene header를 분석해 과거/현재/미래 시점으로 자동 분류

3. **대사 분류 및 그룹화**
   - 캐릭터별로 시점별 대사를 분리 저장

4. **특성 추출**
   - 말투, 성격, 감정 경향, 자주 쓰는 표현 등 총 10개 범주의 특성을 GPT를 통해 추출
   - 장면이 긴 경우 토큰 단위로 청크 분할 및 병합 후 종합 서술 생성

5. **대사 양 분석**
   - 개성 판단이 불가능할 정도로 대사가 적거나 일반적이면 자동으로 저장 후 제외

6. **일관성 평가**
   - 새롭게 입력된 대사가 기존 캐릭터 특성과 얼마나 일치하는지를 평가
     - **기존 특성과의 정합성 평가**
     - **공식 캐릭터 정보와의 정합성 평가**
   - 각각의 평가에서 GPT로 점수화(score + reason) 수행

7. **최종 판단**
   - 시점별로 캐릭터의 대사가 일관성 있는지 판단 (일관성 있음 / 없음 / 정보 부족)
   - 판단 기준은 단순 점수가 아닌 GPT 설명 기반

8. **결과 요약 및 사용자 보고**
   - 일관성이 없는 경우에 대해서만 GPT가 사용자 친화적으로 자연어 보고 생성

##  Prompt Engineering 설계 포인트

- **GPT를 단순 응답 도구가 아닌 판단 로직의 핵심으로 설계**
- 각 단계의 prompt는 역할 기반으로 구분됨 (e.g., extractor, classifier, evaluator, summarizer)
- 불완전하거나 모호한 응답에 대해 재질문 로직을 구성하고, 실패 응답에 대한 graceful fallback 포함
- 불확실성 처리를 위한 자동 재검토 루틴 구성 (예: 이름 추출 불일치, 대사 부족 시 대사 보존 등)

##  결과 저장 구조

- 캐릭터별, 시점별로 특성 분석 결과 저장 (`character_traits_by_time.json`)
- 판단 실패 시 대사 별도 저장 후 후속 업데이트 가능 (`dialogue_storage.json`)
